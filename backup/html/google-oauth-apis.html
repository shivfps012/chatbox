Required Google APIs for OAuth Authentication

To fix the "Error 401: invalid_client" issue, make sure these APIs are enabled in your Google Cloud project.

REQUIRED Google Sign-In API
Core API for OAuth authentication that allows users to sign in with their Google accounts.
Enable API
REQUIRED People API
Provides access to user profile information like name, email, and profile picture. (This replaces the older Google+ API)
Enable API
REQUIRED Google Identity Services API
Core authentication service that handles OAuth 2.0 flows and identity tokens.
Enable API
RECOMMENDED Identity and Access Management (IAM) API
Supports identity management and access control for your project.
Enable API

Additional Steps for Fixing OAuth Issues

1. Verify OAuth Client Configuration

Make sure your OAuth client settings match your environment variables:

  • Go to Credentials
  • Edit your OAuth 2.0 Client ID
  • Verify Authorized JavaScript Origins: http://localhost:5173
  • Verify Authorized Redirect URIs: http://localhost:5000/api/auth/google/callback

2. Check OAuth Consent Screen

Ensure your consent screen is properly configured:

  • Go to OAuth Consent Screen
  • Add your email address as a test user
  • Verify scopes include profile and email
  • Make sure all required fields are filled out

3. Test Your Configuration

After enabling these APIs and verifying settings:

  1. Restart your server: npm run server
  2. Try signing in with Google again
  3. If it fails, check server logs for detailed error messages

Note: After enabling APIs, it may take a few minutes for changes to propagate through Google's systems.
